## 🛠️ How to Use This Repository

Every problem is set up as its own isolated Go module with two main files:
1. `function_name.go`: A template containing the **LeetCode problem description** and an empty function stub. 
2. `function_name_test.go`: A comprehensive set of **perfect test cases** ready to validate your solution.

### Step-by-step Guide:

1. **Pick a problem:** Navigate into any problem folder.
   ```bash
   cd 01_arrays_strings/001_two_sum
   ```
2. **Implement your solution:** Open the `.go` file and replace the `return` statement in the function stub with your actual logic.
3. **Run the tests:** Test your code instantly using Go's built-in testing tool!
   ```bash
   go test -v
   ```
   If your logic is correct, you'll see a glorious `PASS`! 🎉
